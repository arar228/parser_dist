 1 Общая информация  
1.1 Авторизация. Организация взаимодействия  
Партнёр может подключиться  к сервисам API самостоятельно в личном кабинете  (дост упно только 
руководителю) или через запрос на api@ocs.ru , предварительно выполнив следующие действия : 
• получить токен, который потребуется для авторизации при каждом запросе к API;  
• предоставить фиксиров анный список IP -адресов (диапазонов), с которых будет 
осуществляться соединение с сервисом OCS;  
• ознакомит ься с квотами ( ограничениями ) на выполняемые операции;  
• задать уровень доступа (работа с товарным каталогом, доступ к резервированию) . 
Примечание:  При этом партнёр  обязательно д олжен иметь доступ к системе B2 B. 
1.2 Ссылки  
Рабочая версия :

Тестовая версия :

1.3 Ограничения, настраиваемые для партнёр а 
Для партнёр а настраивается ограничение по максимальному количеству запросов в час. Текущие 
настройки  ограничений можно посмотреть в личном кабинете B2 B в разделе «Доступ к API» . 
2 Каталог  (Catalog)  
2.1 Основная предоставляемая функциональность  
Базовая функциональность каталога это:  
• получение информации о товарных категориях;  
• получение информации о наличии товаров и ценах на товар;  
• получение справочной информации.  
 
Примечание:  Поддерживаемый формат обмена – JSON.

2.2.1 Получение информации о товарных категориях  
GET /api/v2/catalog/categories  
Метод предоставляет полное дерев о товарных категорий OCS в следующем виде:  
• Код товарной категории  – category ; 
• Название товарной категории  – name ; 
• Список подкатегорий  – children .

«X-API-Key» ─ токен доступа к API, передается в заголовке запроса:  
curl - X GET "https://connector.b2b.ocs.ru/api/v2/catalog/categories" -H 
"accept: application/json" -H "X-API-Key: api_token"

 2.2.2.1  Входные параметры  
• Категории – categories . Обязательный параметр, представляющий собой список кодов 
товарных категорий, разделенных запятыми, без пробелов (необходимо использовать 
ключевое слово all для запроса по всем доступных товарным категориям);  
• Город отгрузки  – shipmentcity . Обязательный параметр, который выбирается из списка 
допустимы х городов регионального подразделения OCS, с которым работает партнёр. 
Чтобы узнать перечень допустимых городов отгрузки, необходимо воспользоваться 
запросом  Получение информации о допустимых городах отгрузки ; 
• Доступность для заказа  – onlyavailable . Параметр определяет, требуется ли возвращать 
только позиции, доступные для заказа (резервирования). Некоторые товары поставить в 
резерв можно только через менеджера. Данный параметр не используется совместно с 
поиском по отсутствующим на складах позициям ( includemissing=true ). Значение по 
умолчанию – false ; 
• Параметры для выбора категории товаров, которые требуется возвращать в результате 
запроса:  
o includeregular  – определяет, требуется ли возвращать кондиционные товары. 
Значение по умолчанию – true; 
o include sale – определяет, требуется ли возвращать товары по распродаже. Значение 
по умолчанию – false ; 
o Includeuncondition  – определяет, требуется ли возвращать некондиционные товары 
(с дефектами упаковки или поврежденное оборудование). Значение по умолчанию – 
false. 
Примечание:  При запросе необходимо указать как минимум одну категорию 
товаров: кондиционные товары, не кондиция или распродажа (т.е. как минимум один 
из перечисленных выше параметров категории товаров должен иметь значение 
true).  
• Параметр выгрузки ссыл ок на фото некондиционного товара – includeunconditionalimages . 
Данный параметр определяет, требуется ли возвращать ссылки на изображения для 
некондиционного товара. Значение по умолчанию – false . 
• Отсутствующие товары – includemissing . Данный параметр определяет, требуется ли 
возвращать отсутствующие в данный момент на складах позиции. Параметр не 
используется совместно с поиском , только по доступным для заказа позициям 
(onlyavailable=true ). Значение по умолчанию – false . 
• Описание товара  – withdescriptions. Определяет требуется ли показывать описания товаров. 
Может быть использовано для уменьшения объёма передаваемых данных. (значение по 
умолчанию - true). 
• Местоположение остатков  – locations . Данный параметр является фильтром 
местоположен ий остатков по складам. Он позволяет отфильтровать товары только на 
складах в интересующих городах или других местоположениях. Параметр не используется 
при поиске по отсутствующим в данный момент позициям.  
• Производители – producers . Данный параметр является фильтром по производителям. Он 
позволяет отфильтровать товары конкр етных производителей. Параметр не используется 
при поиске по отсутствующим в данный момент позициям.  
Значение по умолчанию – пусто.  
Примечание:  можно ограничить список городов и ви дов транзита, п о которым 
возвращаются остатки.

4 Заказы  (Orders)  
4.1 Основная функциональность  
Базовая функциональность:  
• Создание заказа (резервирование товара);  
• Редактирование заказа (добавление строк, изменение зарезервированного количества, 
снятие резервов, запрос на изменение цены);  
• Удаление строк заказа (снятие резервов);  
• Получение информации о заказах;  
• Передача строк заказа под управление менеджеру (под отгрузку).  
 
При создании и изменении заказов используетс я синхронная и асинхронная схемы работы.  В случае 
асинхронной схемы  запросы валидируются и ставятся в очередь на выполнение (буферизуются). 
Партнёр сразу получает ответ, содержащий код операции, если она успешно принята в обработку. 
Результат выполнения оп ерации необходимо получать по ко ду операции отдельным запросом 
Получение информации о выполнении асинхронной операции .  
В случае использования синхронной схемы  запросы не ставятся в очередь, а начинают выполнение  
немедленно , на партнёрской стороне ответ  приходит немедленно  c результатом по запрошенной 
операции после полного исполнения запроса в цикле обмена. Если ответ на запрос не возвращен, 
то его можно получить позже по коду операции тем же запросом, что и для асинхронной схемы.
