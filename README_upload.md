# Скрипты для копирования проекта на удаленный сервер

Этот набор скриптов позволяет скопировать весь проект `parser_dist` на удаленный сервер по примеру команды:
```bash
scp netlab/netlab_export.py dc-srv@office-srv:~/parser_dist/netlab/
```

## Доступные скрипты

### 1. `upload_all_folders.sh` (Linux/macOS)
Полный bash скрипт для копирования всех папок и файлов проекта.

### 2. `upload_all_folders.ps1` (Windows)
PowerShell версия скрипта для Windows.

### 3. `upload_folders_simple.sh` (Linux/macOS)
Упрощенная версия для быстрого копирования.

## Настройка

Перед использованием отредактируйте параметры в скриптах:
- `REMOTE_USER` - имя пользователя на удаленном сервере
- `REMOTE_HOST` - адрес удаленного сервера  
- `REMOTE_PATH` - путь на удаленном сервере

## Использование

### Linux/macOS
```bash
# Сделать скрипт исполняемым
chmod +x upload_all_folders.sh

# Запустить копирование
./upload_all_folders.sh
```

### Windows
```powershell
# Запустить PowerShell скрипт
.\upload_all_folders.ps1
```

## Что копируется

### Папки:
- `A_TREAD/` - модуль Абсолют Трейд
- `core/` - основные модули (БД, логирование, уведомления)
- `logs/` - логи работы системы
- `marvel/` - модуль Marvel
- `merlion/` - модуль Merlion
- `netlab/` - модуль Netlab
- `ocs/` - модуль OCS
- `resursmedio/` - модуль Ресурс Медио
- `static/` - статические файлы
- `treolan/` - модуль Treolan
- `vvp/` - модуль VVP
- `tests/` - тестовые файлы

### Основные файлы:
- `main.py` - главный файл приложения
- `config.py` - конфигурация
- `api.py` - API интерфейс
- `server.py` - веб-сервер
- Различные скрипты и конфигурационные файлы

## Требования

- SSH доступ к удаленному серверу
- Установленные `scp` и `ssh` команды
- Права на создание папок и файлов на удаленном сервере

## Примечания

- Скрипты создают структуру папок на удаленном сервере автоматически
- Если папка пуста, выводится предупреждение, но процесс продолжается
- Все файлы копируются с сохранением структуры папок
- Логи копирования выводятся в консоль для отслеживания прогресса
